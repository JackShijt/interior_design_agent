> 调研时间：2026-07-14
> 调研范围：主流商业 / 开源 3D 建模与设计软件的技术架构、核心功能与实现原理

## 二、核心建模技术原理

| 技术 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **多边形建模（Polygon）** | 用顶点 + 边 + 面（三角/四边）构成的网格表达形体 | 灵活、硬件友好、实时渲染兼容 | 曲面精度依赖面数，放大有棱角 | 游戏、影视中低模、绝大多数 DCC |
| **NURBS（非均匀有理 B 样条）** | 用控制点 + 节点向量定义数学曲面，可无限细分不失真 | 曲面精度高、光滑、可控 | 拓扑复杂、难做拓扑变化、渲染需转换 | 工业、汽车、航空航天、珠宝 |
| **细分曲面（Subdivision Surface）** | 在多边形控制网格上迭代细分，趋近光滑极限曲面 | 兼顾多边形可控与 NURBS 光滑 | 计算开销、极限面需收敛 | 角色、产品外观 |
| **参数化 / 特征建模** | 以变量 + 约束 + 历史树驱动几何，改参数自动更新 | 可追溯、可编辑、工程友好 | 学习曲线陡、自由造型弱 | CAD、BIM |
| **边界表示（BREP）** | 精确描述物体边界（面/边/顶点）与拓扑关系 | 几何精确、支持分析与制造 | 数据结构复杂 | CAD 内核（Parasolid/ACIS） |
| **构造实体几何（CSG）** | 用布尔运算（并/交/差）组合基本体 | 逻辑清晰、适合工程体 | 难表达有机曲面 | CAD、编程建模 |
| **数字雕刻（Sculpting）** | 类似捏黏土，直接在表面上推拉笔刷 | 极高细节、直观 | 网格重拓扑负担、非参数 | 角色高模、生物 |
| **动态网格（DynaMesh）** | 实时重建均匀拓扑，消除极点 / 非流形 | 起稿自由、不卡拓扑 | 仅 ZBrush 系 | ZBrush 起稿到中模 |
| **程序化 / 节点建模** | 通过可视化节点图定义生成规则 | 可批量、可复现、参数驱动 | 概念门槛高 | 建筑、Houdini 特效 |

## 三、逐款软件调研

### 1. Blender（开源 / 免费）
- **技术架构**：C/C++ 核心 + Python 脚本扩展；模块化（建模、几何节点、雕刻、动画、合成）。
- **核心功能**：多边形 / 雕刻 / 几何节点（程序化）建模；内置两大渲染引擎。
  - **Cycles**：基于物理的离线光线追踪（路径追踪），支持 GPU/CPU。
  - **Eevee**：基于物理的实时光栅化渲染器；2026 推出 **Eevee Next** 升级实时光照 / 阴影管线。
- **实现要点**：开源社区驱动，几何节点系统支持节点化程序建模；雕刻模式借鉴 ZBrush 思路。
- **适用**：独立开发者、学生、小团队；影视 / 动画 / 游戏 / 3D 打印。

### 2. Autodesk Maya
- **技术架构**：C++ 核心，MEL / Python 脚本，节点图依赖系统（DG）。
- **核心功能**：多边形 / NURBS / 细分曲面建模，强大动画与绑定，动力学与特效。
- **实现要点**：以节点网络驱动场景； industry 标准，影视 / 游戏工作室主流。
- **适用**：影视动画、AAA 游戏。

### 3. Autodesk 3ds Max
- **技术架构**：C++ 核心，MAXScript / Python，模块化修改器栈（Modifier Stack）。
- **核心功能**：多边形建模见长，修改器（如涡轮平滑）实现细分曲面；室内 / 建筑可视化、游戏资产。
- **实现要点**：修改器栈顺序执行几何变换，非破坏性编辑。
- **适用**：游戏、室内外设计、建筑可视化。

### 4. Maxon Cinema 4D（C4D）
- **技术架构**：C++ 核心，Python / Coffee 脚本，MoGraph 程序化模块。
- **核心功能**：易用多边形建模 + MoGraph 克隆 / 阵列程序化生成；动态图形（Motion Graphics）强项。
- **适用**：广告、栏目包装、动态设计。

### 5. ZBrush（Maxon）
- **技术架构**：独有 Pixol 技术（记录位置 + 颜色 + 深度 + 材质），C++ 核心。
- **核心功能**：数字雕刻；**DynaMesh** 动态拓扑；**IMM（Insert Multi Mesh）** 笔刷；纳米网格 / 阵列实例化；实时布尔。
- **实现要点**：体素 / 动态网格重拓扑，笔刷直接推拉表面；强调"像捏黏土"。
- **适用**：角色高模、生物、细节雕刻，再烘培到游戏 / 影视流程。

### 6. SolidWorks（Dassault）
- **技术架构**：基于 Parasolid 几何内核（BREP），C++ 核心，VBA / Python API。
- **核心功能**：参数化特征建模（草图 → 拉伸 / 旋转 / 扫描 / 放样 → 特征树）；装配、工程图、仿真。
- **实现要点**：**特征树 / 设计历史**记录每一步，改参数自动重算几何；基于 BREP 精确实体。
- **适用**：机械设计、制造。

### 7. Autodesk Fusion 360
- **技术架构**：云端协同 + 本地客户端，基于 Geomagic / 自研内核，参数化 + 直接建模双模式。
- **核心功能**：参数化建模（时间轴 Timeline 特征链）+ 直接建模；CAM、仿真、PCB 一体。
- **实现要点**：完整保留设计历史树，多模式混合编辑，云同步。
- **适用**：产品工业设计、机械、创客。

### 8. Trimble SketchUp
- **技术架构**：Ruby 脚本扩展，推拉（Push/Pull）直接建模。
- **核心功能**：极简多边形建模，推拉面生成体；大量扩展组件库。
- **实现要点**：面向非专业，强调直觉操作，实时推拉。
- **适用**：建筑、室内、规划快速概念。

### 9. Autodesk Revit（BIM）
- **技术架构**：参数化 BIM 数据库，C++ 核心，Dynamo 可视化编程。
- **核心功能**：建筑信息模型，构件带属性 / 参数，改动联动全模型；自动出图 / 工程量。
- **实现要点**：以数据库驱动几何，参数化族（Family）系统；Dynamo 节点编程扩展。
- **适用**：建筑设计、施工、运维。

### 10. Rhinoceros 3D（Rhino / 犀牛）
- **技术架构**：基于 OpenNURBS 内核（NURBS/BREP），C++ 核心，RhinoScript / Python。
- **核心功能**：精确 NURBS 曲面建模；**Grasshopper** 可视化编程插件。
- **实现要点**：以 NURBS 数学曲面为核心；Grasshopper 用节点图定义参数化算法生成复杂形体；Rhino.Inside 可嵌入 Revit。
- **适用**：工业、珠宝、建筑参数化、船舶。

### 11. SideFX Houdini
- **技术架构**：节点化程序化内核，VEX 语言，C++ 核心。
- **核心功能**：全程序化建模 / 动画 / 特效；节点网络定义生成规则，可批量 / 随机化。
- **实现要点**：以"程序化节点图"而非直接编辑为核心哲学；VEX 高性能脚本。
- **适用**：电影特效、程序化环境、复杂生成式建模。

## 四、技术实现共性总结

1. **几何内核**：CAD 类依赖商业内核（Parasolid、ACIS、OpenNURBS）；DCC 类自研网格系统。
2. **脚本 / 可编程**：几乎全部支持 Python 或自有脚本语言（MEL、MAXScript、VEX、Ruby、Grasshopper）。
3. **渲染分离**：建模软件多内置或外接渲染器（Cycles/Eevee、V-Ray、Redshift、Arnold）。
4. **工作流组合**：影视 / 游戏常用 **Maya + ZBrush + Substance + 引擎** 流水线；建筑常用 **Rhino + Grasshopper + Revit**。
5. **趋势**：开源（Blender）崛起；AI 辅助生成（如文本 / 图像生成 3D）进入工作流；云协同（Fusion 360、云端渲染）。
